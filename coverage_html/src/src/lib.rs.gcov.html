      12                 :             : //! # Key-Value-Storage API and Implementation
      13                 :             : //!
      14                 :             : //! ## Introduction
      15                 :             : //!
      16                 :             : //! This crate provides a Key-Value-Store using [TinyJSON](https://crates.io/crates/tinyjson) to
      17                 :             : //! persist the data. To validate the stored data a hash is build and verified using the
      18                 :             : //! [Adler32](https://crates.io/crates/adler32) crate. No other direct dependencies are used
      19                 :             : //! besides the Rust `std` library.
      20                 :             : //!
      21                 :             : //! The key-value-storage is opened or initialized with [`Kvs::open`] and automatically flushed on
      22                 :             : //! exit by default. This can be controlled by [`Kvs::flush_on_exit`]. It is possible to manually
      23                 :             : //! flush the KVS by calling [`Kvs::flush`].
      24                 :             : //!
      25                 :             : //! All `TinyJSON` provided datatypes can be used:
      26                 :             : //!   * `Number`: `f64`
      27                 :             : //!   * `Boolean`: `bool`
      28                 :             : //!   * `String`: `String`
      29                 :             : //!   * `Null`: `()`
      30                 :             : //!   * `Array`: `Vec<KvsValue>`
      31                 :             : //!   * `Object`: `HashMap<String, KvsValue>`
      32                 :             : //!
      33                 :             : //! Note: JSON arrays are not restricted to only contain values of the same type.
      34                 :             : //!
      35                 :             : //! Writing a value to the KVS can be done by calling [`Kvs::set_value`] with the `key` as first
      36                 :             : //! and a `KvsValue` as second parameter. Either `KvsValue::Number(123.0)` or `123.0` can be
      37                 :             : //! used as there will be an auto-Into performed when calling the function.
      38                 :             : //!
      39                 :             : //! To read a value call [`Kvs::get_value::<T>`](Kvs::get_value) with the `key` as first
      40                 :             : //! parameter. `T` represents the type to read and can be `f64`, `bool`, `String`, `()`,
      41                 :             : //! `Vec<KvsValue>`, `HashMap<String, KvsValue` or `KvsValue`. Also `let value: f64 =
      42                 :             : //! kvs.get_value()` can be used.
      43                 :             : //!
      44                 :             : //! If a `key` isn't available in the KVS a lookup into the defaults storage will be performed and
      45                 :             : //! if the `value` is found the default will be returned. The default value isn't stored when
      46                 :             : //! [`Kvs::flush`] is called unless it's explicitly written with [`Kvs::set_value`]. So when
      47                 :             : //! defaults change always the latest values will be returned. If that is an unwanted behaviour
      48                 :             : //! it's better to remove the default value and write the value permanently when the KVS is
      49                 :             : //! initialized. To check whether a value has a default call [`Kvs::get_default_value`] and to
      50                 :             : //! see if the value wasn't written yet and will return the default call
      51                 :             : //! [`Kvs::is_value_default`].
      52                 :             : //!
      53                 :             : //!
      54                 :             : //! ## Example Usage
      55                 :             : //!
      56                 :             : //! ```
      57                 :             : //! use rust_kvs::{ErrorCode, InstanceId, Kvs, OpenNeedDefaults, OpenNeedKvs, KvsValue};
